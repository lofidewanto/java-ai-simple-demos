🚀 Guest Book Project

Spring Boot Web Application - Complete Summary

📋 Project Goal

Build a Spring Boot web application called "Guest Book" for the demo-sdd-spring project that allows users to view, add, edit, and delete guest book entries through both a web UI and REST API with Swagger documentation.

Core Requirements

  • Features: View, Add, Edit, Delete operations (CRUD)
  • UI Approach: Both REST API and server-side rendered Thymeleaf templates
  • Data Model: Guest book entries with name, email, message, and timestamps (created_at, updated_at)
  • Database: H2 in-memory database
  • Documentation: OpenAPI/Swagger UI for API documentation
  • Styling: Bootstrap 5 for modern UI
  • Display Order: Entries displayed newest first
  • Architecture: NO service interface - only direct service class implementation

🔌 REST API Endpoints

  • GET /api/entries - Get all entries
  • GET /api/entries/{id} - Get entry by ID
  • POST /api/entries - Create new entry
  • PUT /api/entries/{id} - Update entry
  • DELETE /api/entries/{id} - Delete entry

🚀 Running the Application

Prerequisites

  • Java 21 or higher
  • Maven 3.6+

Build & Run

# Build the project
mvn clean install

# Run the application
mvn spring-boot:run

# The app starts on http://localhost:8080

Running Tests

# Run all tests
mvn test

# Run specific test class
mvn test -Dtest=GuestBookServiceTest

Stop Application

pkill -f "spring-boot:run"
